# Guardia evals

Safety rules for Guardia, evaluated by Autoval. Each file is a single test case.

## Schema

```json
{
  "id": "snake_case_identifier",
  "name": "Human-readable name shown in the dashboard checklist",
  "description": "One-sentence plain-English description of the rule",
  "input": "User prompt to send to Guardia",
  "assertions": [
    { "type": "must_contain", "value": "..." },
    { "type": "must_not_contain", "value": "..." },
    { "type": "must_contain_any", "value": ["...", "..."] }
  ],
  "evidence": {
    "source": "URL or doc reference for why this rule exists",
    "finding": "Short summary of the evidence"
  },
  "category": "operational | drug_interaction | allergen | dosing | recall | certification",
  "created_by": "baseline | autoval autonomous scan | autoval manual trigger"
}
```

## Lifecycle

- Baseline rules (`created_by: "baseline"`) are hand-authored — operational invariants Guardia already meets.
- Discovered rules (`created_by: "autoval ..."`) are generated by the Autoval agent when it finds a bad output in production logs (ClickHouse) that violates a previously-unknown safety constraint.
- Both run on every PR to `prompts/system-prompt.txt`. The PR must pass ALL rules before merge.
